# 1.单例模式

最佳实践是采用枚举方式（JVM提供线程安全、防止反射攻击，防止序列化破坏。）

为了单例创建时机可控，使用静态内部类，防止反射攻击.setAccessible(true)，可以在构造器里加已存在，再new就直接抛异常

饿汉式的两种方式方式

1.私有化构造器直接new一个

2.枚举方式

public enum Singleton {
    uniqueInstance;
}

懒汉式的四种实现方式

1.一个if，线程不安全

2.synchronized锁

3.双重校验锁，两个if

4.内部类实现，JVM提供线程安全，volatile防止指令重排，构造器判断防止反射攻击，readResolve()防止序列化破坏

public class Singleton {
    
    private static boolean flag = false;
    
    private Singleton(){
        //处理反射攻击
        synchronized(Singleton.class){
            if(flag == false){
                flag = !flag;
            }else{
                throw new RuntimeException("单例模式，对象已经创建，反射不可以再创建")
            }
        }
    }
    
    public static class SingletonHandler{
        private volatile static final Singleton INSTANCE = new Singleton();
    }
    
    public static Singleton getInstance(){
        return SingletonHandler.INSTANCE;
    }
    
    //实现readResolve方法可以解决反序列化攻击
    //反序列化时会检查有没有这个方法，如果有可以调用这个方法返回对象
    private Object readResolve() {
        return InnerHolder.INSTANCE;
    }
}

# 2.简单工厂模式

提供一个类来集中管理对象创建的细节，让客户类和具体子类解耦，只用继承简单工厂类传入参数，具体创建过程交由该类实现

简单工厂模式 (opens new window)

# 3.工厂方法模式

定义了一个创建对象的接口，但由子类决定要实例化哪个类。工厂方法把实例化操作推迟到子类。

与简单工厂相比，在简单工厂中，创建对象的是另一个类，而在工厂方法中，是由子类来创建对象。

工厂方法模式 (opens new window)

# 4.抽象工厂模式

创建一连串的相关对象，工厂方法只是用于创建一个对象

抽象工厂使用多个工厂方法创建一连串对象

客户类直接使用抽象工厂类的方法获取多个实例

抽象工厂模式 (opens new window)

# 5.模板方法

拥有相同的算法结构，定义相同的部分算法（方法），不同的算法（方法）交由子类实现

Spring IOC 容器对bean的初始化使用的refresh()方法就是一个典型的模板方法

模板方法也称钩子方法

模板方法 (opens new window)
