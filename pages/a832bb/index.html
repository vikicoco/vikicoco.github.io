Spring笔记

# Spring

# 1. Spring简介

  • Spring开源的IOC【DI】和AOP容器框架
  • Spring也是一家公司，官网：https://spring.io/

# 2. Spring特点

  • 非侵入式：无需实现某个接口，或继承某个类
    • Servlet:侵入式，必须实现某个结构或继承某个类
  • 组件化：即插即用
  • 一站式：使用Spring可以解决【持久化层-业务逻辑层-控制层】三层问题。
  • IOC：Inversion of Control【反转控制】，将对象的控制权反转spring
  • AOP：Aspect-Oriented Programming【面向切面编程】

# 3. 搭建Spring框架步骤【★】

  • 导入jar包
    • 将springframework添加到本地仓库的org目录下即可
  • 新建Student【测试类】
  • 创建Spring核心配置文件【applicationContext.xml或beans.xml】
    • 位置：resources目录下
    • 将Student类装配Spring容器中
  • 创建Spring容器对象【ClassPathXmlApplicationContext】
    • 通过容器对象获取Student对象

# 4. getBean()三种方式

  • getBean(String id)：从spring容器中，通过id获取对象

    • 不足：默认获取对象类型是Object，需要强转，比较繁琐

  • getBean(Class clazz)：从spring容器中，通过Class获取对象

    • 不足：当容器中有多个相同Class对象时，会报如下错误：

      expected single matching bean but found 2: ltq,zzq

  • getBean(String id,Class clazz)：从spring容器中，通过id和Class获取对象

    • 建议使用

# 6. Spring中IOC【DI】

  • IOC全称：Inversion of Control，反转控制。【将对象控制权反转给Spring容器】

  • DI(Dependency Injection)，依赖注入。

    • 总结: IOC 就是一种反转控制的思想， 而DI是对IOC的一种具体实现。

  • Spring中有IOC思想，IOC思想必须基于IOC容器来完成，而IOC容器在最底层实质上就是一个对象工厂.

    • Spring实现IOC两种方式
      • BeanFactory【Spring底层实现】
      • ApplicationContext【推荐使用】

  • ApplicationContext结构图

    • ApplicationContext
      • ConfigurableApplicationContext
        • ClassPathXmlApplicationContext【基于类路径下，检索xml文件】
        • FileSystemXmlApplicationContext【基于系统路径（真实路径）下，检索xml文件】

# 9. Spring中FactoryBean

  • 工厂bean设置的类型可以和返回的类型不一样，其返回的类型通过该工厂bean的getObject方法指定。

  • 示例代码

    package com.viki.pojo;

import org.springframework.beans.factory.FactoryBean;

public class MyFactoryBean implements FactoryBean<Student> {
    @Override
    public Student getObject() throws Exception {
        Student student = new Student(101,"zq");
        System.out.println("student 被创建啦！" + student);
        return student;
    }

    @Override
    public Class<?> getObjectType() {
        return Student.class;
    }

//    设置是否为单例
    @Override
    public boolean isSingleton() {
        return true;
    }
}

# 10. Spring中bean作用域【★】

设置bean作用域语法,在bean标签中添加属性:scope，scope属性取值范围如下：

  1. singleton：单例，在Spring的IOC容器中仅存在一个Bean实例
    • spring在创建容器对象时，创建bean对象【全局只创建一次容器对象，所以全局只有一个bean对象】
  2. prototype：多例，每次调用getBean方法都返回一个新的Bean实例
    • 每次调用getBean()方法时，都会创建一个bean对象
  3. request：请求域
    • 当前请求有效，离开当前请求失效
    • 当前请求：地址栏不变即为当前请求，地址栏改变不在当前请求
  4. session：会话域
    • 当前会话有效，离开当前会话失效
    • 当前会话：浏览器不关闭&不更换即为当前会话

# 11. Spring中bean的生命周期

# 11.1 标准生命周期

① 通过构造器或工厂方法创建bean实例 ② 为bean的属性设置值和对其他bean的引用 ③ 调用bean的初始化方法 ④ bean可以使用了 ⑤ 当容器关闭时，调用bean的销毁方法

# 11.2 添加后置处理器，bean的生命周期

① 通过构造器或工厂方法创建bean实例 ② 为bean的属性设置值和对其他bean的引用 将bean实例传递给bean后置处理器的postProcessBeforeInitialization()方法 ③ 调用bean的初始化方法 将bean实例传递给bean后置处理器的postProcessAfterInitialization()方法 ④ bean可以使用了 ⑤ 当容器关闭时，调用bean的销毁方法

# 12. Spring中自动装配【★】

Spring提供两种装配方式

  1. 手动装配：在bean中添加set注入或者构造注入等方式
  2. 自动装配：无需set注入或构造注入，就可以自动装配属性值方式

  • 语法

    • 在bean标签中，添加autowire属性，常用属性值如下：

      • byName：通过类中属性名称与ioc容器中id匹配；如匹配成功，则自动装配成功，如未匹配成功，则会将null值装配

      • byType：通过类中属性类型与ioc容器中class匹配，匹配结果如下：

        • 匹配失败

          • 匹配0个：不会报错，会将null值装配

          • 匹配多个：会报错

            expected single matching bean but found 2: address,address2

        • 匹配成功

          • 匹配1个bean时，则匹配成功

  • 注意

    • 自动装配底层使用set注入
    • 实战中建议使用byName或byType呢？
      • 答：都不建议使用，建议使用注解方式

# 13. Spring中注解【★★】

约束 > 配置【注解】 > 代码

使用注解为了简化配置【0配置】

使用注解准备

  • 开启组件扫描
  • 导入aopjar包

# 13.1 使用spring注解，简化装配bean配置

1) @Component

标识一个受Spring IOC容器管理的普通组件

2) @Repository

标识一个受Spring IOC容器管理的持久化层组件

3) @Service

标识一个受Spring IOC容器管理的业务逻辑层组件

4) @Controller

标识一个受Spring IOC容器管理的表述层控制器组件

  • 使用注解默认会将类名首字母小写作为bean的id
  • 可以使用value属性为注解，设置bean的id值【如只书写一个属性时，value可省略】

# 13.2 使用spring注解，简化装配bean中的属性配置【注入数值】

  • 语法：@Autowired

  • 作用：使用@Autowired注解实现自动装配，装配规则如下：

    • 先按照byType方式进行匹配，

      • 如匹配成功【容器中有且仅有一个匹配类型】，则自动装配成功

      • 如匹配失败，再按照byName进行匹配，如匹配成功则正常使用，如匹配失败，则会按照byType方式提示错误信息

        • 匹配0个：

          • @Autowired中required属性作用：设置装配属性是否必须自动装配

            • required=true：必须为指定属性自动装配
            • required=false：不必须为指定属性自动装配

          • required=true【默认值】，会报错

             expected at least 1 bean which qualifies as autowire candidate. Dependency annotations: {@org.springframework.beans.factory.annotation.Autowired(required=true)}
            1

          • required=false，不会报错，会将null值装配

        • 匹配多个

          • 会报错：expected single matching bean but found 2: address,address2

  • 注意：

    • 默认@Autowired注解书写在字段中，底层反射方式注入数值
    • @Autowired注解书写在setXXX()方法中，底层使用set注入
    • @Autowired注解书写在构造方法中，底层使用构造注入

  • @Qualifier注解

    • 将ioc容器中bean的id自动装配到指定类的属性中
    • 一般与@Autowired注解配合使用

  • @Value注解，自动装配字面量数值

    • 示例代码

      @Value("1001")
private Integer id;
@Value("zs")
private String stuName;

# 17.1 动态代理【设计模式之一】

  • 生活中案例
    • 中介是我们租房时，代理对象
    • 中介和我们“兄弟”关系
    • 相关名词：横向提取、动态织入
    • 总结：代理对象与目标对象，不能相互强制转换

# 17.2 实现计算器功能分析

  • 将日志功能，书写在业务代码中发现问题如下：

    1) 代码混乱：越来越多的非业务需求(日志)加入后，原有的业务方法急剧膨胀。每个方法在处理核心逻辑的同时还必须兼顾其他多个关注点。

    • 业务代码与非业务代码书写在一处时，比较混乱

    2) 代码分散: 只是为了满足这个单一需求，就不得不在多个模块（方法）里多次重复相同的日志代码。如果日志需求发生变化，必须修改所有模块。

    • 同一个非业务功能，书写在多个业务代码中

  • 解决问题

    • 先将非业务代码“横向提取”
    • 再“动态织入”回业务代码中

# 17.3 使用动态代理实现计算器代理对象

  • 实现思路

    • Proxy：代理类的基类
      • Proxy.newProxyInstance()：创建代理对象
    • InvocationHandler：执行目标对象的代理方法
      • invoke()

  • 实现步骤

    1. 创建目标对象【被代理代码】
    2. 创建获取代理对象方法
    3. 提供有参构造器

# 17.4 示例代码

@Component
public class CalcProxy {

    //目标对象
    private Object target;

    public CalcProxy(Object object){
        this.target = object;
    }

    //创建获取代理对象方法
    public Object getProxy(){
        //代理对象
        Object proxyObj = null;
        //创建代理对象
        //参数：类加载器、所有接口Class、InvocationHandler
        //获取类加载器
        ClassLoader classLoader = target.getClass().getClassLoader();
        //获取目标对象所有接口Class
        Class<?>[] interfaces = target.getClass().getInterfaces();
        //InvocationHandler【执行目标对象的代理方法】
        proxyObj = Proxy.newProxyInstance(classLoader, interfaces, new InvocationHandler() {
            /*
                proxy
                method:执行方法Method对象
                Object[]:参数
             */
            @Override
            public Object invoke(Object proxy, Method method, Object[] args) throws Throwable {
                //获取方法名
                String methodName = method.getName();
                //执行业务代码之前，织入日志
                MyLogging.methodBefore(methodName,args);
                //执行目标对象，相应方法
                Object rs = method.invoke(target, args);
                //执行业务代码之后，织入日志
                MyLogging.methodAfter(methodName,rs);
                return rs;
            }
        });
        return proxyObj;
    }
}

# 18.2 AOP简介

  • AOP全称：Aspect-Oriented Programming，面向切面编程

  • AOP是OOP扩展或补充

    • OOP：面向对象编程，纵向继承机制
    • AOP：面向切面编程，横向提取机制

  • AOP术语

    • 横切关注点：非核心业务称之为横切关注点

    • 切面：将横切关注点提取类中，类称之为切面

    • 通知：将横切关注点提取到类中后，横切关注点更名为：通知

    • 目标：目标对象【CalcImpl】

    • 代理：代理对象

      •   @Autowired
  private Calc calc;	// com.sun.proxy.$Proxy20
        1
        2

    • 连接点：先将横切关注点提取切面类中【通知】，再动态将通知织入回业务代码中；织入位置称之为，连接点。【织入之前】

    • 切入点：先将横切关注点提取切面类中【通知】，再动态将通知织入回业务代码中；织入位置称之为，连接点。【织入之后】

  • 环绕通知【前四种通知集合】

    • 语法：@Around

    • 作用：将业务代码【前置|后置|返回|异常】通知，整合一处

    • 注意：环绕通知的参数，只能ProceedingJoinPoint。

    • 示例代码

      @Around(value = "myPointCut()")
public Object methodAround(ProceedingJoinPoint pjp){
    Object rs = null;
    //获取方法名
    String methodName = pjp.getSignature().getName();
    //获取参数
    Object[] args = pjp.getArgs();
    try {
        //前置通知
        System.out.println("日志功能【前置通知】：执行"+methodName+"()方法之前，参数："+ Arrays.toString(args));
        //执行目标对象的目标方法【add()】
        rs = pjp.proceed();
        //返回通知
        System.out.println("日志工能【返回通知】：执行"+methodName+"()方法,返回结果rs:"+rs);
    } catch (Throwable ex) {
        ex.printStackTrace();
        //异常通知
        System.out.println("日志工能【异常通知】：执行"+methodName+"()方法,出现异常ex:"+ex);
    } finally {
        //后置通知
        System.out.println("日志功能【后置通知】：执行"+methodName+"()方法之后，参数："+ Arrays.toString(args));
    }
	return rs;
}

# 23.4 Spring声明式事务管理中属性

  • 事务传播行为

    • 语法：propagation=Propagation.REQUIRED【默认值】

    • 概述：当事务方法被另一个事务方法调用时，必须指定事务应该如何传播。例如：方法可能继续在现有事务中运行，也可能开启一个新事务，并在自己的事务中运行。

    • 7种事务传播行为

      传播属性 描述
      REQUIRED 如果有事务在运行，当前的方法就在这个事务内运行；否则就启动一个新的事务，并在自己的事务内运行。
      REQUIRES_NEW 当前的方法****必须****启动新事务，并在自己的事务内运行；如果有事务正在运行，应该将它挂起。
      SUPPORTS 如果有事务在运行，当前的方法就在这个事务内运行，否则可以不运行在事务中。
      NOT_SUPPORTED 当前的方法不应该运行在事务中，如果有运行的事务将它挂起
      MANDATORY 当前的方法必须运行在事务中，如果没有正在运行的事务就抛出异常。
      NEVER 当前的方法不应该运行在事务中，如果有正在运行的事务就抛出异常。
      NESTED 如果有事务正在运行，当前的方法就应该在这个事务的嵌套事务内运行，否则就启动一个新的事务，并在它自己的事务内运行。

  • 事务隔离级别

    • 语法：isolation=Isolation.

    • 概述：一个事务与其他事务隔离的程度称为隔离级别。

    • 四种隔离级别

      脏读 不可重复读 幻读
      READ UNCOMMITTED【1】 有 有 有
      READ COMMITTED【2】 无 有 有
      REPEATABLE READ【4】 无 无 有

  • 事务超时

    • timeout：超时事务属性可以设置事务在强制回滚之前可以保持多久。这样可以防止长期运行的事务占用资源。

  • 事务只读

    • readonly：只读事务属性可以设置这个事务只读取数据但不更新数据, 这样可以帮助数据库引擎优化事务。

  • 事务异常回滚【不回滚】

    • noRollbackFor：设置指定不回滚的异常类型

# 23.5 基于xml方式配置声明式事务

<?xml version="1.0" encoding="UTF-8"?>
<beans xmlns="http://www.springframework.org/schema/beans"
       xmlns:xsi="http://www.w3.org/2001/XMLSchema-instance"
       xmlns:context="http://www.springframework.org/schema/context"
       xmlns:aop="http://www.springframework.org/schema/aop" xmlns:tx="http://www.springframework.org/schema/tx"
       xsi:schemaLocation="http://www.springframework.org/schema/beans http://www.springframework.org/schema/beans/spring-beans.xsd http://www.springframework.org/schema/context https://www.springframework.org/schema/context/spring-context.xsd http://www.springframework.org/schema/aop https://www.springframework.org/schema/aop/spring-aop.xsd http://www.springframework.org/schema/tx http://www.springframework.org/schema/tx/spring-tx.xsd">

    <!--引入外部属性文件-->
    <context:property-placeholder location="classpath:druid.properties"></context:property-placeholder>

    <!--配置数据源-->
    <bean id="dataSource" class="com.alibaba.druid.pool.DruidDataSource">
         <property name="username" value="${jdbc.username}"></property>
        <property name="password" value="${jdbc.password}"></property>
        <property name="url" value="${jdbc.url}"></property>
        <property name="driverClassName" value="${jdbc.driverClassName}"></property>
        <property name="initialSize" value="${jdbc.initialSize}"></property>
        <property name="maxActive" value="${jdbc.maxActive}"></property>
    </bean>

    <!--配置JdbcTemplate-->
    <bean id="jdbcTemplate" class="org.springframework.jdbc.core.JdbcTemplate">
        <!--配置数据源属性-->
        <property name="dataSource" ref="dataSource"></property>
    </bean>

    <!--配置BookShopDaoImpl-->
    <bean id="bookShopDao" class="com.viki.spring.tx.xml.BookShopDaoImpl">
        <property name="jdbcTemplate" ref="jdbcTemplate"></property>
    </bean>

    <!--配置BookShopServiceImpl-->
    <bean id="bookShopService" class="com.viki.spring.tx.xml.BookShopServiceImpl">
        <property name="bookShopDao" ref="bookShopDao"></property>
    </bean>

    <!--配置事务管理器-->
    <bean id="transactionManager" class="org.springframework.jdbc.datasource.DataSourceTransactionManager">
        <!--配置数据源属性-->
        <property name="dataSource" ref="dataSource"></property>
    </bean>
    <!--配置声明式事务-->
    <tx:advice id="tx" transaction-manager="transactionManager">
        <!--设置添加事务的方法-->
        <tx:attributes>
            <!--设置查询的方法的只读属性为true-->
            <tx:method name="find*" read-only="true"/>
            <tx:method name="get*" read-only="true"/>
            <tx:method name="purchase" propagation="REQUIRES_NEW" isolation="READ_COMMITTED"></tx:method>
        </tx:attributes>
    </tx:advice>
    <!--AOP配置-->
    <aop:config>
        <!--配置切入点表达式-->
        <aop:pointcut id="pointCut"          expression="execution(*com.viki.spring.tx.xml.BookShopServiceImpl.purchase(..))"/>
        <!--将事务方法和切入点表达式关联起来-->
        <aop:advisor advice-ref="tx" pointcut-ref="pointCut"></aop:advisor>
    </aop:config>
</beans>
