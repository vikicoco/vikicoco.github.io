lambda表达式

# Lambda表达式

Lambda表达式是一种新的语法糖，让Java开始支持“函数式编程”的语法。

Java是面向对象的编程语言，所有的操作几乎都是基于对象，“一切皆对象”。

如果某个变量或形参是引用数据类型，那么必须给他赋值一个对象再使用。

在开发中，有时候我们并不关心这个对象，我们关心的是这个对象的某个方法的功能，方法体代码，其实就是函数的实现。按照面向对象的编程思想，为了传递这段代码，还要new一个对象，就显得有点啰嗦。

函数式编程思想：把函数当成数据传递

# 函数式接口

函数式接口，是SAM接口（Single Abstract Method）有唯一的抽象方法的接口。

JDK8后可以有default方法，不影响，仍是函数式接口

Lambda表达式只针对“函数式接口”使用。

@FunctionalInterface
public interface Runnable {
    //只有一个抽象方法，并且标识FuncationalInterface
    public abstract void run();
}

# lambda语法

语法结构：

(Lambda表达式的形参列表)-> {Lambda体} (参数类型 参数名称)->{代码语句}

说明：

1.->：称为箭头操作符，或Lambda操作符,由减号和大于号组成，中间不能有空格，表示指向动作 2.(Lambda表达式的形参列表)，就是函数式接口的抽象方法的形参列表

例如：

对于Rannable接口中的run()方法，则Lambda表达式的形参列表为 ()->{} 对于Comparator<T>接口中的compare(T t1, T t2)方法，则Lambda表达式的形参列表为 (T t1, T t2)->{}

作用：

Lambda表达式的作用是给函数式接口的变量/形参赋值的

Lambda表达式的简化

（1）当{Lambda体}只有一个语句，可以省略{}和语句后面的; ps:如果此时是return语句，要把return也省略 （2）当（Lambda表达式的形参列表）的形参列表的形参类型是已知的，或者是可以根据泛型自动推断的，那么类型可以省略 （3）如果（Lambda表达式的形参列表）的形参列表的类型省略了，并且形参只有一个，那么()可以省略

​ 如果形参列表是多个参数，()不能省略

​ 如果形参列表是空参()，那么()不能省略

lambda的简化例子

//Runnable接口中run()方法无参的，()不能省略，只有一条语句，；和 {} 和 return可以省略
new Thread(()-> System.out.println("hello")).start();

ps：为什么Thread可以使用lambda表达式？

在Thread的构造函数中，public Thread(Runnable target)传入的Runnable接口是函数式接口(JDK1.8)

//使用lambda标准格式
list.forEach((element) -> {
    System.out.print(element);
});

//Comsumer函数式接口中void accept(T t)方法中参数只有一个，()可以是省略，只有一条语句，；和 {} 和 return可以省略
//Comsumer接口是消费型接口，抽象方法是`void accept(T t); 参数类型是t
list.forEach(element-> System.out.print(element));

//对lambda再次简化，使用方法引用::遍历,下文讲述
list.forEach(System.out::print);

# 方法引用

语法格式：

类名/对象名::方法名

说明：

::双冒号被称为引用运算符

如果lambda函数体中的代码方案已经在某个方法中实现了，就可以直接使用双冒号::来引用该方法替代lambda表达式

满足条件：

1.{lambda体}中只有一个语句，并且这个语句是调用现有的对象/类的方法来完成。

2.lambda表达式的(形参列表)中的形参，要么作为调用方法的对象，要么作为调用方法的实参，所 有形参都要用上

3.在整个{lambda体}中，没有使用到除lambda表达式中额外的数据/对象等

方法引用例子：

//例如：
Arrays.sort(arr, (t1, t2) -> t1.compareToIgnoreCase(t2));

//1.{lambda体}中只有一个语句 return t1.compareToIgnoreCase(t2);
//2.Lambda表达式的(形参列表)是(t1, t2)，t1是作为调用compareToIgnoreCase方法的对象，t2是作为调用compareToIgnoreCase方法的实参。
//3.没有用到额外数据/对象

//所以可以使用方法引用，因为t1是String对象，compareToIgnoreCase()是String类的方法
Array.sort(arr,String::compareToIgnoreCase)

# 总结

1.使用lambda表达式的前提条件

传入的参数接口必须是函数式接口

2.简化lambda表达式使用方法引用::的前提条件

1.{lambda体}中只有一个语句，并且这个语句是调用现有的对象/类的方法来完成。

2.lambda表达式的(形参列表)中的形参，要么作为调用方法的对象，要么作为调用方法的实参，所 有形参都要用上

3.在整个{lambda体}中，没有使用到除lambda表达式中额外的数据/对象等
